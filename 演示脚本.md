# 香辣卤味管理系统 — 演示脚本

## 演示前准备

1. 确认 SQL Server 正在运行
2. 启动后端：`cd phase2-backend && mvnw.cmd spring-boot:run`
3. 启动前端：`cd phase3-frontend && npm run dev`
4. 打开浏览器 → http://localhost:5173

---

## 第一幕：顾客端（约 3 分钟）

### 1.1 浏览商品
| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 打开 http://localhost:5173 | 看到商品卡片网格、公告栏、分类筛选 |
| 2 | 点击分类下拉选「卤味荤菜」 | 只显示荤菜类商品 |
| 3 | 点击「套餐组合」 | 看到 6 个套餐 |
| 4 | 点击「全部」清除筛选 | 恢复全部 19 个商品 |

### 1.2 商品详情
| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 5 | 点击「招牌辣鸭脖」 | 进入详情页，显示单价¥18、库存500、辣度🌶🌶🌶🌶 |
| 6 | 查看底部用户评价 | 看到 5 条中文评价（鸭脖非常入味...） |

### 1.3 注册与登录
| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 7 | 点击右上角「登录」 | 进入登录页 |
| 8 | 切换到「注册」Tab | 显示用户名、密码、邮箱表单 |
| 9 | 输入：用户名=demouser, 密码=test123, 邮箱=demo@test.com | |
| 10 | 点击「注册」按钮 | 提示"注册成功"，自动跳转首页，右上角显示"demouser" |

### 1.4 购物车与下单
| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 11 | 点击「招牌辣鸭脖」→ 数量改为3 →「加入购物车」 | 提示"已添加 3 x 招牌辣鸭脖到购物车" |
| 12 | 回到首页，点击「卤藕片」→ 数量2 →「加入购物车」 | 跳转购物车 |
| 13 | 查看购物车 | 2件商品：鸭脖x3(¥54) + 藕片x2(¥16) = ¥70 |
| 14 | 点击「去结算」 | 进入结算页 |
| 15 | 选优惠券（如有）→ 选积分 500 → 备注写"多加辣" | 显示折扣和实付金额 |
| 16 | 点击「提交订单」 | 提示"下单成功"，显示订单号和应付金额 |

### 1.5 模拟支付
| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 17 | 点击「模拟支付」 | 提示"支付成功"，自动跳转订单页 |
| 18 | 查看订单列表 | 刚下的订单状态为"已确认" |

## 第三幕：亮点展示（约 1 分钟）

| 亮点 | 演示方式 |
|------|----------|
| 🌐 前后端分离 | 浏览器 F12 → Network，展示 API 请求/响应 JSON |
| 🔐 JWT 认证 | 展示请求头 `Authorization: Bearer eyJ...` |
| 📱 响应式 | 浏览器缩小到手机宽度，卡片自动适配单列 |
| 🛒 购物车持久化 | 刷新页面后购物车数据仍在 |
| 🎫 优惠券+积分 | 结算页同时使用优惠券和积分，自动计算折扣 |
| ⚡ 行级库存锁 | 说明并发下单时库存不会超卖 |
