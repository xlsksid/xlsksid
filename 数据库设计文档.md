# 香辣卤味管理系统 — 数据库设计文档

## 一、概念结构设计

### 1.1 实体及属性

#### 用户 (User)
```
用户编号、用户名、密码哈希、邮箱、手机号、角色、头像、账户余额、积分余额、软删除标记、创建时间、更新时间
```

#### 商品分类 (Category)
```
分类编号、分类名称、描述、排序权重、分类图片、软删除标记、创建时间、更新时间
```

#### 商品 (Product)
```
商品编号、商品名称、描述、售价、成本价、库存、单位、图片、辣度(1-5)、上架标记、软删除标记、创建时间、更新时间
```

#### 订单 (Order)
```
订单编号、用户编号、订单号、总额、优惠金额、抵扣积分数、积分抵扣金额、实付金额、状态、备注、软删除标记、创建时间、更新时间
```

#### 订单明细 (OrderDetail)
```
明细编号、订单编号、商品编号、数量、单价快照、小计、创建时间
```

#### 支付记录 (Payment)
```
支付编号、订单编号、用户编号、金额、支付方式、支付状态、交易流水号、支付完成时间、创建时间、更新时间
```

#### 评价 (Feedback)
```
评价编号、用户编号、商品编号、订单编号、评分(1-5)、评价内容、软删除标记、创建时间、更新时间
```

#### 积分记录 (PointsRecord)
```
记录编号、用户编号、积分变动(+/-)、类型、描述、关联订单编号、创建时间
```

#### 优惠券模板 (CouponTemplate)
```
模板编号、名称、类型、折扣率、满减金额、最低消费、有效天数、总量、已发量、启用标记、创建时间、更新时间
```

#### 用户优惠券 (UserCoupon)
```
优惠券编号、用户编号、模板编号、状态、使用订单号、有效期起止、使用时间、创建时间
```

### 1.2 实体间关系

```
用户 ──1:N──▶ 订单          （一个用户可下多个订单）
用户 ──1:N──▶ 支付记录      （一个用户可有多条支付记录）
用户 ──1:N──▶ 评价          （一个用户可评价多次）
用户 ──1:N──▶ 积分记录      （一个用户可有多条积分变动）
用户 ──1:N──▶ 用户优惠券    （一个用户可持有多张优惠券）

分类 ──1:N──▶ 商品          （一个分类下有多个商品）

商品 ──1:N──▶ 订单明细      （一个商品可被多次购买）
商品 ──1:N──▶ 评价          （一个商品可被多次评价）

订单 ──1:N──▶ 订单明细      （一个订单包含多个明细）
订单 ──1:1──▶ 支付记录      （一个订单对应一条支付记录）
订单 ──1:N──▶ 评价          （一个订单可被评价多次）
订单 ──1:N──▶ 积分记录      （一个订单可关联多条积分记录）

优惠券模板 ──1:N──▶ 用户优惠券 （一个模板可发放给多个用户）
```

## 三、关键约束与索引

### 3.1 CHECK 约束

| 表 | 约束 | 说明 |
|----|------|------|
| users | role IN ('admin','staff','customer') | 三角色 |
| users | balance ≥ 0, points ≥ 0 | 非负 |
| products | price/cost/stock ≥ 0 | 非负 |
| products | spiciness BETWEEN 1 AND 5 | 辣度范围 |
| orders | total/discount/actual ≥ 0 | 非负 |
| orders | status 7 种状态值 | 状态机 |
| order_details | quantity > 0, subtotal ≥ 0 | 正数量 |
| payments | payment_method 5 种 | 支付方式 |
| payments | payment_status 4 种 | 支付状态 |
| feedback | rating BETWEEN 1 AND 5 | 1-5 星 |
| coupon_template | 互斥约束：discount_rate 与 reduction_amount | 二选一 |
| user_coupon | status 3 种 | 状态 |

### 3.2 核心索引

| 表 | 索引 | 类型 | 用途 |
|----|------|------|------|
| users | IX_users_role | 筛选 | WHERE is_deleted=0 |
| products | IX_products_category | 普通 | 按分类查询 |
| products | IX_products_search | 复合 (category_id, spiciness, is_available) | 搜索筛选 |
| orders | IX_orders_user_created | 复合 (user_id, created_at DESC) | 我的订单 |
| orders | IX_orders_status | 筛选 | 后台订单管理 |
| user_coupon | UQ_user_coupon_one_active | 唯一筛选 | 同一用户同一模板仅一张未使用券 |

### 3.3 订单状态机

```
pending ──▶ confirmed ──▶ preparing ──▶ delivering ──▶ completed
   │                                                       ▲
   └──────────▶ cancelled                                  │
   └──────────▶ refunded ──────────────────────────────────┘
```
